Sistema de diagnostico mediante voz de una PC en relacion a sus status de cpu, memoria, disco, red y procesos en curso mediante AI usando el API de OpenAi o  groq.
----------------------------------------------------------------------------------------------------------------------------------------------
Este sistema busca facilitar el analisis del funcionamiento de un computador en base a un escaneo del sistema. De esta manera tanto personas sin conocimiento profundo sobre computaciones como expertos pueden utilizarlo como un primer contacto hacia un equipo que este presentando problemas.

Voice-Based PC Diagnostic System Using AI (CPU, Memory, Disk, Network, and Running Processes)
----------------------------------------------------------------------------------------------------------------------------------------------
This system aims to simplify the analysis of a computer’s performance through a system scan. In this way, both users without deep technical knowledge and experts can use it as a first point of contact when dealing with a computer that is experiencing issues.
----------------------------------------------------------------------------------------------------------------------------------------------
- Initially, the program starts by displaying an interface:
<img width="591" height="619" alt="main" src="https://github.com/user-attachments/assets/84163fec-5a89-4fa6-80a3-cc3b9b1617d8" />

- After that, we press the "Start Diagnostic" button. Once this is done, we can see at the bottom of the interface a list of the system characteristics of our PC:
<img width="590" height="617" alt="diagnostic ready" src="https://github.com/user-attachments/assets/9781d28b-5a27-4bd2-91e0-0161cf7921e4" />

- Next, we can press the "Activate AI" button so that the program starts listening. By saying the keyword "System", we will receive a confirmation message at the bottom, and we can then explain to the program the issue we are experiencing.
  Once the user finishes speaking, the AI sends the system diagnostic data, the user’s reported problem, a brief process check (performed using an Excel-based process list), and other specific instructions to the API.
  As a result, the user receives a detailed report about their system along with the main recommendations provided by the AI.
